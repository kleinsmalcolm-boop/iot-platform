⚡ IoT Platform API

Connect any device to your IoT platform in minutes. Send sensor data, receive commands, and monitor everything in real time.

Introduction

The IoT Platform API lets your devices send sensor data to the cloud and receive commands from your dashboard. It works with any device that can make HTTP requests — Arduino, ESP32, ESP8266, Raspberry Pi, and more.

Base URL: http://192.168.2.11:3000

Quick Start

Get your first device sending data in 4 steps:

1

Create an account

Register at POST /api/auth/register with your email and password.

2

Register a device

Call POST /api/my-devices — you'll get back an API key automatically.

3

Copy your API key

It looks like iot_a3f9b2c1d4e5f6... — keep it safe, your device needs it.

4

Send your first reading

POST to /api/device/data with your API key in the header. Done!

Authentication

There are two types of authentication depending on who is making the request:

WhoMethodHeader
Users / DashboardJWT TokenAuthorization: Bearer <token>
IoT DevicesAPI KeyX-API-Key: iot_abc123...
Never share your API key publicly. If it gets leaked, regenerate it from your dashboard immediately.

Send Sensor Data

Your device calls this endpoint to send a single sensor reading. Use the API key you got when registering the device.

POST /api/device/data

Send a single sensor reading from your device.

Headers

HeaderValue
X-API-KeyYour device API key
Content-Typeapplication/json

Request Body

// Single reading
{
  "pin":   "temperature",
  "value": 25.6
}

Response

{
  "success": true,
  "device":  "My Arduino"
}

Example — curl

curl -X POST http://192.168.2.11:3000/api/device/data \
  -H "X-API-Key: iot_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"pin":"temperature","value":25.6}'

Send Multiple Readings

Send several sensor readings in one request. Much more efficient than making a separate request for each sensor — use this when your device has multiple sensors.

POST /api/device/data

Send multiple readings at once using the data array.

Request Body

// Multiple readings in one request
{
  "data": [
    { "pin": "temperature", "value": 25.6 },
    { "pin": "humidity",    "value": 60.2 },
    { "pin": "motion",      "value": 1 }
  ]
}

Receive Commands

Your device polls this endpoint to check if your dashboard sent it a command — like turning on an LED or changing a threshold value.

GET /api/device/commands

Get pending commands for this device. Commands are marked as delivered after retrieval.

Response

{
  "commands": [
    {
      "id":      1,
      "command": "LED_ON",
      "value":   "1"
    }
  ]
}
Poll this endpoint every 10 seconds in your device loop. Avoid polling faster than every 5 seconds to stay within rate limits.

Register a Device

Creates a new device under your account and automatically generates an API key for it.

POST /api/my-devices

Requires user JWT token in Authorization header.

Request Body

{
  "name": "Living Room Sensor",
  "type": "esp32"
}

Response

{
  "id":         3,
  "name":       "Living Room Sensor",
  "device_key": "dev_a3f9b2c1d4",
  "api_key":    "iot_a3f9b2c1d4e5f6789abc",
  "type":       "esp32",
  "online":     false
}

List Your Devices

GET /api/my-devices

Returns all devices registered to your account including their API keys.

Get Device Readings

GET /api/my-devices/:id/readings?limit=50

Returns historical sensor readings for a device. Default limit is 50, max 500.

Send Command to Device

Send a command from your dashboard to a device. The device will receive it next time it polls /api/device/commands.

POST /api/my-devices/:id/command

Requires user JWT token. Device receives command on next poll.

Request Body

{
  "command": "LED_ON",
  "value":   "1"
}

Built-in Commands

Arduino / ESP32 SDK

The easiest way to connect your ESP32 or Arduino to the platform. Download the SDK, fill in your WiFi and API key, and upload. Your device will start sending data immediately.

Required Libraries

Install these in Arduino IDE via Tools → Manage Libraries:

LibraryPurpose
WiFiBuilt-in with ESP32 board package
HTTPClientBuilt-in with ESP32 board package
ArduinoJsonSearch "ArduinoJson" by Benoit Blanchon

Configuration

After downloading, open the file and change these 4 lines at the top:

const char* WIFI_SSID     = "YOUR_WIFI_NAME";
const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
const char* API_KEY       = "iot_YOUR_API_KEY";
const char* SERVER_URL    = "http://192.168.2.11:3000";
